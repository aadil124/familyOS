## Documentation Rules

- Keep planning documents in Markdown.
- Avoid implementation details until the relevant architecture document is created.
- Keep terminology consistent with the Project Blueprint.
- Update related documents when product scope, architecture, or contracts change.
- Prefer diagrams and tables where they improve clarity.
